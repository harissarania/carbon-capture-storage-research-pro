## 🔄 Agile Approach

This project adopts a **hybrid Agile-Scrum framework** tailored for research and engineering work. Each **sprint spans 4 weeks**, aligned with project phases and key milestones. The backlog is organized into **6 Epics** corresponding to the 6 project phases, with user stories written from the perspective of the role that owns the outcome.

```
  PROJECT LIFECYCLE  →  AGILE MAPPING
  ─────────────────────────────────────────────────────────────
  Phase 1  Initiation & Planning    →  Sprint 1          (Mo 1–2)
  Phase 2  Bench-Scale R&D          →  Sprints 2–4       (Mo 3–8)
  Phase 4  Geological Storage       →  Sprints 3–6       (Mo 6–14)  ← parallel
  Phase 3  Pilot Plant Scale-Up     →  Sprints 5–8       (Mo 9–15)
  Phase 5  HSE & Compliance         →  Sprints 6–8       (Mo 11–16)
  Phase 6  Closeout & Reporting     →  Sprints 8–9       (Mo 15–18)
```

**Scrum Cadence:**
- 📅 **Daily standup** — 9:00 AM, 15 min, core team
- 🗓️ **Sprint review** — End of each 4-week sprint, with Sponsor
- 🔁 **Sprint retrospective** — End of each sprint, core team
- 📊 **Backlog refinement** — Week 3 of each sprint
- 🎯 **Velocity target** — 25–35 story points per sprint

```

```

### How the Team Uses Jira

- **Daily standup** — team members update their CPB item statuses before the 9:00 AM standup; blockers flagged with a comment
- **Sprint planning** — PM creates a new sprint in Jira, drags backlog items in, assigns story points and owners
- **Sprint review** — completed items demonstrated; status moved to DONE; velocity calculated
- **Backlog refinement** — PM and leads refine upcoming stories (acceptance criteria, estimates) in Week 3 of each sprint
- **Risk linkage** — high-risk work items (R-01, R-04, R-07) are linked to corresponding Jira issues for traceability

## ✅ Definition of Done

A backlog item is considered **Done** only when ALL of the following criteria are met:

```
  [ ]  Work completed to specification per acceptance criteria
  [ ]  Peer reviewed by at least one other team member
  [ ]  Data / output uploaded to SharePoint with correct naming convention
  [ ]  Quality control check passed (where applicable)
  [ ]  PM notified and Jira status updated to DONE
  [ ]  Any related risks updated in the risk register
  [ ]  No open non-conformances linked to this item
```
