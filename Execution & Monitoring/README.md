## ✅ Quality Management Plan

### Purpose

The QMP defines how all project deliverables — technical, documentary, and process-related — will be assured and controlled to meet defined standards throughout the 18-month CCS Research Initiative.

**Applicable standards:** ISO 9001:2015 &nbsp;|&nbsp; ISO 27914:2017 &nbsp;|&nbsp; ASTM D1622 &nbsp;|&nbsp; GLP &nbsp;|&nbsp; IEC 61882 (HAZOP) &nbsp;|&nbsp; ISO 14001:2015 &nbsp;|&nbsp; PMBOK 7th Ed.

| # | Objective | Target |
|---|-----------|--------|
| 1 | CO₂ capture efficiency at pilot scale | **≥ 90%** |

### 🔬 Quality Assurance Activities

#### Experimental Data Quality
```
 ┌────────────────────────────────────────────────────────────┐
 │  1. Calibrate all instruments before each campaign         │
 │     └─ Calibration certificates recorded in SharePoint     │
 │  2. Written SOP approved by R&D Lead before each run       │
 │  3. Duplicate / triplicate measurements for critical data  │
 │  4. Raw data → tamper-evident lab notebooks                │
 │     └─ Digitised within 24 hours                          │
 │  5. Independent data check by second team member          │
 └────────────────────────────────────────────────────────────┘
```

#### Document Quality
- All deliverable documents peer-reviewed by at least one other team member before submission
- Version control enforced: `[Code]_[Type]_v[X.X]_[YYYYMMDD]`
- Phase gate reports reviewed by Sponsor before approval
- Final feasibility report undergoes **external peer review** (academic partner)

#### Engineering Quality
- Pilot plant design reviewed by independent EPC contractor before construction
- HAZOP conducted by certified facilitator with full core team attendance
- All commissioning punch list items closed before trial operations begin
- Pressure vessels and pipework inspected to local regulatory standards

## ⚠️ Non-Conformance Process

A **non-conformance (NC)** is any situation where a deliverable, process, or result does not meet the defined quality standard.

### NC Severity Classification

| Class | Definition | Response Time | Escalation |
|-------|-----------|---------------|-----------|
| 🟢 **Minor** | No impact on project objectives | 5 business days | Team lead |
| 🟠 **Major** | Affects deliverable quality | 48 hours | Project Manager |
| 🔴 **Critical** | Affects safety or regulatory compliance | Immediate | Sponsor + HSE Officer |

### NC Resolution Workflow

```
  ┌─────────────┐
  │  1. IDENTIFY │  Any team member records NC in SharePoint NC Log
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │ 2. CLASSIFY  │  PM classifies: Minor / Major / Critical
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │ 3. INVESTIGATE│  Root cause analysis within 48 hours by relevant lead
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │  4. CORRECT  │  Corrective action implemented; work re-done or re-tested
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │  5. VERIFY   │  PM or Lead verifies correction is effective
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │   6. CLOSE   │  NC closed in log with evidence; lessons captured
  └─────────────┘

  ⚠️  Critical NCs → escalated to Sponsor + HSE Officer → stop-work assessment
```

## 📝 Status Reporting

### Report Overview

The **Monthly Status Report** is issued on the **1st Monday of each month** by the Project Manager to the Sponsor, core team, and HSE Officer. Report ID format: `SR-00X`.

### RAG Status Framework

| Colour | Meaning | Action Required |
|--------|---------|----------------|
| 🟢 **Green** | On track — no issues | None |
| 🟡 **Amber** | At risk — action in place | Monitor closely; update next report |
| 🔴 **Red** | Off track — immediate action required | Escalate to Sponsor; corrective plan within 48h |

### Report Structure

| Section | Content |
|---------|---------|
| **1. Overall RAG Status** | Scope / Schedule / Budget / Risk — one RAG per dimension |
| **2. Executive Summary** | 2–3 sentence high-level project health summary for Sponsor |
| **3. Key Achievements** | Deliverables and milestones completed in the reporting period |
| **4. Schedule Status** | Milestone tracker — Planned vs Forecast vs Actual dates |
| **5. Budget Status** | Spend-to-date vs plan by phase; variance % |
| **6. Active Risks & Issues** | Top risks with level, trend, and mitigation update |
| **7. Planned Activities** | Work planned for the next reporting period |
| **8. Decisions Required** | Items needing Sponsor approval, with deadlines |
| **9. Issues Log** | Open issues with owner and resolution status |
| **10. Sign-Off** | PM + Sponsor signatures |
